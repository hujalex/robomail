  ---
  DNS Setup on Namecheap

  Inbound — add MX records

  CloudMailin provides these when you create an address target. Go to your
  CloudMailin dashboard → your address → "DNS Settings" and add the MX records
  they give you. Typically:

  ┌──────┬──────┬──────────────────────┬──────────┐
  │ Type │ Host │        Value         │ Priority │
  ├──────┼──────┼──────────────────────┼──────────┤
  │ MX   │ @    │ alt1.cloudmailin.net │ 10       │
  ├──────┼──────┼──────────────────────┼──────────┤
  │ MX   │ @    │ alt2.cloudmailin.net │ 20       │
  └──────┴──────┴──────────────────────┴──────────┘

  Outbound — add SPF + DKIM + DMARC TXT records

  CloudMailin gives you the exact values in their dashboard under "Sending
  Domains":

  ┌──────┬────────────────────────┬──────────────────────────────────────────┐
  │ Type │          Host          │                  Value                   │
  ├──────┼────────────────────────┼──────────────────────────────────────────┤
  │ TXT  │ @                      │ v=spf1 include:_spf.cloudmailin.net ~all │
  ├──────┼────────────────────────┼──────────────────────────────────────────┤
  │ TXT  │ cloudmailin._domainkey │ (DKIM key from CloudMailin dashboard)    │
  ├──────┼────────────────────────┼──────────────────────────────────────────┤
  │ TXT  │ _dmarc                 │ v=DMARC1; p=none;                        │
  └──────┴────────────────────────┴──────────────────────────────────────────┘

  ---
  CloudMailin Dashboard Setup

  Inbound:
  1. Create an "Address" and set format to JSON (normalized)
  2. Set the target URL to https://your-app.vercel.app/webhooks/inbound
  3. Enable HMAC signing — copy the secret, that becomes INBOUND_WEBHOOK_SECRET
  4. Under "Domain", add your Namecheap domain and verify it

  Outbound:
  1. Go to "SMTP Credentials" — get host, port, username, password
  2. Go to "Sending Domains" — add your domain, copy the DKIM record to
  Namecheap

⏺ You're past the onboarding screen — click Skip This and then navigate within
  the CloudMailin dashboard to find everything:

  INBOUND_WEBHOOK_SECRET

  1. Go to your address (27561b67ba35df6f2dd1@cloudmailin.net) settings
  2. Click "Target" or "Edit Target"
  3. There will be a section called "HTTP POST" with a "Signature Key" or
  "Secret" — that value is your INBOUND_WEBHOOK_SECRET

  If it's not visible there, go to Account → API Credentials or the address
  detail page and look for an HMAC/signature secret.

  SMTP Credentials (outbound)

  1. In the left sidebar click "SMTP Credentials" (or go to Account → SMTP)
  2. You'll see:
    - Host: smtp.cloudmailin.com
    - Port: 587
    - Username: your account username
    - Password: generated password

  Copy those four values into your .env and Vercel env vars:

  CLOUDMAILIN_SMTP_HOST=smtp.cloudmailin.com
  CLOUDMAILIN_SMTP_PORT=587
  CLOUDMAILIN_SMTP_USER=<from dashboard>
  CLOUDMAILIN_SMTP_PASS=<from dashboard>
  INBOUND_WEBHOOK_SECRET=<signature key from target settings>

  One note: that address (27561b67ba35df6f2dd1@cloudmailin.net) is a CloudMailin
   subdomain address. To receive mail at your own domain (e.g.
  agent@yourdomain.com), you still need to go into the address settings and add
  your custom domain with the MX records pointed at CloudMailin.
